discovers damage patterns, recommends similar claims, and detects fraudulent

## Pipeline Components

| # | Component | Technique | Description |
|---|-----------|-----------|-------------|
| 1 | **Data Generation** | Synthetic resampling + fraud injection | Resamples source claims to ~2,000 with 3-tier fraud (blatant/moderate/subtle) + phantom parts |
| 2 | **Part Matching** | Fuzzy (token_sort_ratio) + TF-IDF Cosine | Cross-source part name matching between surveyor and garage records |
| 3 | **Association Rules** | Apriori → confidence/lift filtering | Discovers frequently co-occurring damaged parts in claims |
| 4 | **Recommendations** | TruncatedSVD + Cosine Similarity | Finds similar historical claims for a given claim ID |
| 5 | **Fraud Detection** | Isolation Forest + Random Forest | Dual approach: unsupervised anomaly detection + supervised classifier with GridSearchCV |
| 6 | **Visualisation** | Matplotlib + Seaborn (Agg backend) | 6 auto-generated charts saved to `outputs/plots/` |

## Quick Start

### 1. Install Dependencies

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS/Linux

pip install -r requirements.txt
```

### 2. Generate Data (if starting fresh)

```bash
python generate_data.py
```

This creates `surveyor_data.csv`, `garage_data.csv`, and `Primary_Parts_Code.csv`
in the `data/` directory from the source `garage.csv`.

If you have your own real datasets, place them directly in `data/` with the expected
column names (see [Configuration](#configuration)).

### 3. Train

```bash
python train.py
```

This runs the full 6-step pipeline:
1. Load & clean data
2. Merge datasets
3. Part matching (Fuzzy + TF-IDF Cosine)
4. Association rule mining (Apriori)
5. Recommendation engine (SVD)
6. Fraud detection (Isolation Forest + Random Forest with GridSearchCV)

All models are saved to `models/`, metrics to `outputs/evaluation_metrics.json`,
and plots to `outputs/plots/`.

### 4. Predict (Interactive)

```bash
python predict.py
```

Options:
1. **Find similar part** — enter a part name, get fuzzy + cosine matches
2. **Get claim recommendations** — enter a claim ID, get top-5 similar claims
3. **Check for fraud** — enter an amount + part count, get fraud verdict

## Training & Evaluation Strategy

### Cross-Validation: RepeatedStratifiedKFold

The fraud detection model uses **RepeatedStratifiedKFold** cross-validation
(default: **5 folds × 3 repeats = 15 total evaluations**) to provide robust,
variance-aware performance estimates. This is more reliable than a single
train/test split, especially for imbalanced fraud data (15% positive rate).

### Hyperparameter Fine-Tuning: GridSearchCV

When `GRID_SEARCH_ENABLED = True` in `config.py`, the pipeline automatically
runs **GridSearchCV** over the parameter grid:

| Parameter | Search Space |
|-----------|-------------|
| `n_estimators` | 200, 300, 500 |
| `max_depth` | 8, 12, 16, None |
| `min_samples_split` | 2, 5, 10 |

The best combination is selected by maximising the cross-validated F1 score.
The tuned model is then retrained on the full training set.

### Evaluation Metrics

- **Test Set (80:20 stratified split):** Accuracy, Precision, Recall, F1, ROC-AUC
- **Cross-Validation (15 folds):** Mean ± std for all metrics, plus per-fold scores
- **Unsupervised (Isolation Forest):** Silhouette score, Calinski-Harabasz index

## Configuration

All hyperparameters are centralised in `config.py`:

### Data & Paths

| Parameter | Default | Description |
|-----------|---------|-------------|
| `SAMPLE_SIZE` | `None` | Sample rows after merge (`None` = full data) |
| `TEST_SPLIT_RATIO` | `0.2` | Train/test split ratio (80:20) |
| `RANDOM_STATE` | `42` | Reproducibility seed |

### Part Matching

| Parameter | Default | Description |
|-----------|---------|-------------|
| `FUZZY_THRESHOLD` | `80` | Minimum fuzzy match score (0–100) |
| `COSINE_THRESHOLD` | `0.15` | Minimum TF-IDF cosine similarity |

### Association Rules

| Parameter | Default | Description |
|-----------|---------|-------------|
| `APRIORI_MIN_SUPPORT` | `0.01` | Minimum item-set support |
| `APRIORI_MIN_CONFIDENCE` | `0.1` | Minimum rule confidence |
| `APRIORI_MIN_LIFT` | `1.0` | Minimum rule lift |

### Recommendation Engine

| Parameter | Default | Description |
|-----------|---------|-------------|
| `SVD_N_COMPONENTS` | `10` | SVD latent dimensions |
| `SVD_TOP_N` | `5` | Number of recommendations |

### Fraud Detection

| Parameter | Default | Description |
|-----------|---------|-------------|
| `ISO_FOREST_CONTAMINATION` | `0.10` | Expected anomaly fraction |
| `ISO_FOREST_N_ESTIMATORS` | `200` | Isolation trees count |
| `RF_N_ESTIMATORS` | `300` | Random Forest trees (fallback if GridSearch disabled) |
| `RF_MAX_DEPTH` | `12` | Max tree depth (fallback) |

### Cross-Validation & Grid Search

| Parameter | Default | Description |
|-----------|---------|-------------|
| `CV_N_SPLITS` | `5` | Folds per CV repeat |
| `CV_N_REPEATS` | `3` | Number of CV repetitions |
| `GRID_SEARCH_ENABLED` | `True` | Enable/disable GridSearchCV |
| `RF_PARAM_GRID` | `{...}` | Parameter grid for hyperparameter search |
